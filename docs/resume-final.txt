三、项目经验
--------------------------------------------------------------------------------

智慧社区微服务管理平台
2024.12 - 至今 ｜ 独立开发
技术栈：Spring Boot + Spring Cloud Alibaba + MyBatis-Plus +
        MySQL + Redis + RabbitMQ + Docker

面向社区物业管理的微服务项目，我主要负责活动报名、停车管理两条核心业务链路
以及 AI 智能服务模块的设计与开发，独立完成了从数据库设计到接口实现的全流程。

1. 活动报名库存优化：高并发扣减库存导致数据库锁竞争和超卖问题。改为 Redis 预扣
   库存 + RabbitMQ 异步落库，将瞬时流量前移到缓存和消息队列层。压测 200 并发下
   无超卖，QPS 较直接操作数据库提升约 4 倍。

2. 停车计费幂等控制：道闸重复抓拍存在重复计费风险。在计费接口中引入 Redis 分布
   式锁做幂等控制，同时调整了加锁与事务开启的顺序，减少数据库连接占用时长。重复
   上报场景下无重复扣费发生。

3. 缓存设计与性能优化：对车位余量查询等高频接口使用 Spring Cache + Redis 做缓存，
   配合定时任务做缓存预热。压测中接口平均响应时间从 292ms 降至 3ms 以内，QPS
   显著提升。

4. 系统稳定性保障：通过 Nacos 实现配置动态管理，基于 Sentinel 在网关层做限流和
   服务降级，配合定时任务处理超时未支付订单，保障高并发下核心链路稳定运行。

5. AI 智能客服（探索实践）：基于 Spring AI 调用 DeepSeek，实现了关键字与向量结合
   的混合检索，将社区资料拼入 Prompt 生成回答，设计了无资料拒绝和回答校验机制，
   避免模型编造信息。对 RAG 架构和大模型工程化落地有了基本理解。

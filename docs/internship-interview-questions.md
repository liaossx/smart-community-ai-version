## 一、Java 基础

**知识点 1**：面向对象三大特性
**题**：封装、继承、多态分别是什么？多态怎么实现的？

封装就是把一个类的属性和方法 封装起来  对外只保留一个接口 调用这个接口不需要关注 里面操作实际是怎么样的。 继承就子类复用父类的代码 避免重复造轮子。多态就是 一个接口可以有多个实现类  调用方法时 只需要关注接口有哪几个方法  不需要关注真正实现业务的实现类。在设计角度上，多态是一种解耦的设计 ，接口层定需求，实现类写实现

**知识点 2**：String
**题**：String、StringBuilder、StringBuffer 的区别？String 为什么是不可变的？

String是不可变的，如果对sting对象进行修改，就是直接new一个对象。StringBuilder是直接对对象本体进行修改，不会创建新的对象。因为String 每次进行字符串的拼接 都会new一个string对象 如果出现在循环代码中  就会创建很多string对象占用堆内存 。StringBuilder就是为了解决这种问题，然后StringBuffer跟StringBulider的区别就是加了个锁

**知识点 3**：HashMap
**题**：HashMap 的底层结构？1.7 和 1.8 有什么变化？为什么链表长度到 8 转红黑树？

hashmap的底层结构是数组+链表。通过计算key值 然后找到数组下标 空着就插进去  不是空的挂在链表。1.7和1.8最大的区别就是更改了链表的插入方式。1.7使用的是头插法，扩容的时候遇到冲突会导致死循环，1.8变成了尾插法，解决了死循环的问题，但是遇到冲突可能会造成数据覆盖。还有就是1.8引入了红黑树。但如果要线程安全还是用 ConcurrentHashMap

**知识点 4**：ArrayList vs LinkedList
**题**：ArrayList 和 LinkedList 的区别？什么场景用哪个？

arraylist 是动态数组  linkedlist是 双向链表  。动态数组 的查询比较快 但是 插入和删除比较慢 因为插入一个元素之后  后面的位置都需要挪动。双向链表的查询速度慢，但是插入和删除速度快 只需要修改两个位置的指针就行了（但是前提是先找到插入或者删除的位置，这个时间也是O(n)）。一般场景 都使用arraylist，少部分 头尾频繁增删的场景才会使用

**知识点 5**：异常
**题**：Exception 和 Error 的区别？RuntimeException 和 checked exception 有什么区别？

Exception 是一般是代码 或者参数 出问题了 可以通过try-catch去处理问题 然后error就是 这个系统出问题了  比如jvm堆内存 或者是连接池

**知识点 6**：反射
**题**：什么是反射？在项目中哪里用到了？

反射就是在运行时去jvm的方法去找到这个类的字段和方法，然后去调用它执行方法，不是编译的时候就写死。spring框架下的常用注解都用到了 反射的机制 比如@autowire  @transcationl

## 四、Redis

**知识点 19**：缓存穿透 / 击穿 / 雪崩
**题**：缓存穿透、击穿、雪崩分别是什么？怎么解决？

穿透就是请求访问不存在的数据，缓存层找不到 然后打到数据库。如果遇到大量恶意请求，数据库会挂掉。击穿是一个热点key 突然过期了，然后大量的请求来数据库里找。雪崩就是一系列热点key同时过期，数据库瞬时压力过大。缓存穿透通常采用 缓存一个null值或者空值，设置最短过期时间，把数据格挡在缓冲层。还有一种办法是用布隆过滤器，提前把主键id注入，然后把不符合的请求全部拦住。缓存击穿的话，设置定时任务，每几分钟就去写缓存  。 缓存雪崩的话，设置不同的过期时间，还有定时任务。

## 五、消息队列（RabbitMQ）

**知识点 24**：消息可靠性
**题**：RabbitMQ 怎么保证消息不丢失？

发送端要使用confirm机制确认消息已经发出，然后消息队列里 开启消息持久化，防止网络挂掉 消息丢失，消费端开启手动ack机制，确认收到消息才删除队列里的信息

**知识点 25**：重复消费
**题**：消息重复消费了怎么办？你怎么处理的？

可以用唯一索引防重，或者做定时任务兜底

**知识点 26**：使用场景
**题**：什么场景用消息队列？你项目里怎么用的？

高并发，高写入场景下使用消息队列，为了减少数据库的写入压力，把数据延迟写入数据库。我项目中的 活动报名就使用了这个，先用redis做预扣减 然后再异步写入数据库。可以增加用户的体验，不用花长时间等待。

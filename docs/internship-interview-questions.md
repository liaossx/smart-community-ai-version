## 一、Java 基础

**知识点 1**：面向对象三大特性
**题**：封装、继承、多态分别是什么？多态怎么实现的？

封装就是把一个类的属性和方法 封装起来  对外只保留一个接口 调用这个接口不需要关注 里面操作实际是怎么样的。 继承就子类复用父类的代码 避免重复造轮子。多态就是 一个接口可以有多个实现类  调用方法时 只需要关注接口有哪几个方法  不需要关注真正实现业务的实现类。在设计角度上，多态是一种解耦的设计 ，接口层定需求，实现类写实现

**知识点 2**：String
**题**：String、StringBuilder、StringBuffer 的区别？String 为什么是不可变的？

String是不可变的，如果对sting对象进行修改，就是直接new一个对象。StringBuilder是直接对对象本体进行修改，不会创建新的对象。因为String 每次进行字符串的拼接 都会new一个string对象 如果出现在循环代码中  就会创建很多string对象占用堆内存 。StringBuilder就是为了解决这种问题，然后StringBuffer跟StringBulider的区别就是加了个锁

**知识点 3**：HashMap
**题**：HashMap 的底层结构？1.7 和 1.8 有什么变化？为什么链表长度到 8 转红黑树？

hashmap的底层结构是数组+链表。通过计算key值 然后找到数组下标 空着就插进去  不是空的挂在链表。1.7和1.8最大的区别就是更改了链表的插入方式。1.7使用的是头插法，扩容的时候遇到冲突会导致死循环，1.8变成了尾插法，解决了死循环的问题，但是遇到冲突可能会造成数据覆盖。还有就是1.8引入了红黑树。但如果要线程安全还是用 ConcurrentHashMap

**知识点 4**：ArrayList vs LinkedList
**题**：ArrayList 和 LinkedList 的区别？什么场景用哪个？

arraylist 是动态数组  linkedlist是 双向链表  。动态数组 的查询比较快 但是 插入和删除比较慢 因为插入一个元素之后  后面的位置都需要挪动。双向链表的查询速度慢，但是插入和删除速度快 只需要修改两个位置的指针就行了（但是前提是先找到插入或者删除的位置，这个时间也是O(n)）。一般场景 都使用arraylist，少部分 头尾频繁增删的场景才会使用

**知识点 5**：异常
**题**：Exception 和 Error 的区别？RuntimeException 和 checked exception 有什么区别？

Exception 是一般是代码 或者参数 出问题了 可以通过try-catch去处理问题 然后error就是 jvm或者是系统层面出问题了，代码处理不了

**知识点 6**：反射
**题**：什么是反射？在项目中哪里用到了？

反射就是在运行时去jvm的方法去找到这个类的字段和方法，然后去调用它执行方法，不是编译的时候就写死。spring框架下的常用注解都用到了 反射的机制 比如@autowire  @transcationl

## 三、MySQL

**知识点 12**：索引数据结构
**题**：MySQL 索引底层为什么用 B+ 树？和二叉树、Hash 比有什么好处？

二叉树如果递增插入的会退化成链表，hash不支持范围查找。b+树 是自平衡树，矮胖树，非叶子节点只存索引，然后叶子节点用双向链表串联起来，可以范围查询。

**知识点 13**：聚簇索引和二级索引
**题**：聚簇索引和二级索引的区别？什么是回表？

聚簇索引也是一级索引也是主键索引。二级索引是非聚簇索引，也是唯一索引，普通索引，联合索引。一级索引的b+树的叶子节点存储整行数据，二级索引的叶子节点存储着索引列的值和主键id，需要通过主键id回表查询自己想要的数据，这就是回表

**知识点 14**：最左前缀
**题**：联合索引 (a, b, c)，WHERE b=1 AND c=2 走索引吗？为什么？

索引会失效，因为根据最左前缀原则，查询条件需要a在最左边的时候索引才能生效，如果跳过最左边的做查询条件 索引会失效。建立联合索引的时候需要查询频率较高，等值比较的放左边，范围查询的放右边

**知识点 15**：索引失效
**题**：索引失效的场景有哪些？LIKE '%xxx' 为什么不走索引？

1，没有遵循最左前缀原则，2，左模糊查询 3，查询条件的字段和索引的字段类型不一致 4，查询条件中有不等于 !=

## 四、Redis

**知识点 19**：缓存穿透 / 击穿 / 雪崩
**题**：缓存穿透、击穿、雪崩分别是什么？怎么解决？

穿透就是请求访问不存在的数据，缓存层找不到 然后打到数据库。如果遇到大量恶意请求，数据库会挂掉。击穿是一个热点key 突然过期了，然后大量的请求来数据库里找。雪崩就是一系列热点key同时过期，数据库瞬时压力过大。缓存穿透通常采用 缓存一个null值或者空值，设置最短过期时间，把数据格挡在缓冲层。还有一种办法是用布隆过滤器，提前把主键id注入，然后把不符合的请求全部拦住。缓存击穿的话，设置定时任务，每几分钟就去写缓存  。 缓存雪崩的话，设置不同的过期时间，还有定时任务。

**知识点 20**：数据结构
**题**：Redis 有哪几种常用数据结构？你项目里用了哪些？
有string list  set  hash  zset  redis全局是一个map<key,value>  前面这些数据结构 是关于value的。我项目中查询车位缓存用了string的数据结构，活动防重报名 用了set数据结构
**知识点 21**：分布式锁
**题**：怎么用 Redis 实现分布式锁？SET NX 是什么意思？锁过期了怎么办？

用set key value nx ex 10 nx表示key不存在才创建 。ex 表示最大过期时间是10。使用lua脚本去释放锁  先get拿到自己的value 然后才del  避免误删别人的锁。 使用reddsion的看门狗机制 续锁

**知识点 22**：持久化
**题**：RDB 和 AOF 的区别？生产环境两个都开吗？
RDB是 是定时快照，文件小恢复快。AOF是记录写命令日志，会记录每个写操作，文件大但是恢复慢。生产环境两个都开RDB做主备份

**知识点 23**：缓存一致性
**题**：先删缓存再更新数据库，还是先更新数据库再删缓存？为什么？

先更新数据库再删缓存，因为如果先删缓存再更新数据库的话 这段时间有空窗，有新的请求再更新数据库之前访问了 那么就会生成旧数据的缓存

## 五、消息队列（RabbitMQ）

**知识点 24**：消息可靠性
**题**：RabbitMQ 怎么保证消息不丢失？

发送端要使用confirm机制确认消息已经发出，然后消息队列里 开启消息持久化，防止网络挂掉 消息丢失，消费端开启手动ack机制，确认收到消息才删除队列里的信息

**知识点 25**：重复消费
**题**：消息重复消费了怎么办？你怎么处理的？

可以使用唯一索引防重，比如什么停车订单管理，可以用订单号做唯一索引，从数据库层防止重复数据写入。在业务层可以用分布式锁，保证同一件事只有一个线程可以去操控 避免重复创建订单

**知识点 26**：使用场景
**题**：什么场景用消息队列？你项目里怎么用的？

高并发，高写入场景下使用消息队列，为了减少数据库的写入压力，把数据延迟写入数据库。我项目中的 活动报名就使用了这个，先用redis做预扣减 然后再异步写入数据库。可以增加用户的体验，不用花长时间等待。

**知识点 27**：选型
**题**：RabbitMQ 和 Kafka 有什么区别？什么时候用哪个？
RabbitMq是消息队列，消息消费完就没了。kafka是分布式日志(一般需要在分布式的场景下，性能才会彻底释放，吞吐百万级每秒，机器挂了不丢数据)，消息可以回溯重放，kafka靠硬盘顺序写吞吐极高，适合大数据管道。rabbitmq轻量，部署简单。项目里一天消息不到万级，rabbitmq完全够用。

## 六、微服务

**知识点 28**：Nacos 的作用
**题**：Nacos 是干什么的？注册中心和配置中心分别解决什么问题？

nacos在项目中有两个作用 注册中心————服务启动时把自己注册上去，其他服务通过服务名而不是ip来调用，实现系统内部的软编码；还有就是做配置中心--把系统的核心配置都托管到nacos上，修改配置不用重启服务。

**知识点 29**：Gateway
**题**：Spring Cloud Gateway 是干什么的？和 Nginx 有什么区别？

gateway是微服务体系的统一入口，负责路由转发，集成sentinel做接口限流，鉴权。nginx是通过反向代理，通常放在最外层做负载均衡和静态资源。nginx是基础的设施 而gateway是配合nacos专门为微服务 做的服务 。

**知识点 30**：Sentinel
**题**：Sentinel 是干什么的？限流和熔断有什么区别？

sentinel 是做流量控制，有两个关键的就是接口限流和熔断机制。限流和熔断的区别就是限流是流量大了就拒绝服务，熔断就是下层的服务出问题了，就挂掉，走其他的方法。  在我的项目中，接口限流是在gateway层做的，然后配置信息存储在nacos中。

**知识点 31**：Feign / OpenFeign
**题**：Feign 的原理是什么？一个接口加 @FeignClient 为什么就能调用了？

fegin的原理就是发送一个http请求去访问这个接口 ，然后调用这个方法 到自己的模块，使用@feignclient 注解 会使用负载均衡去nacos里面找到匹配的实例 ，然后再自动拼装http请求 发送请求

## 七、JVM

**知识点 32**：堆结构
**题**：JVM 内存模型大概讲一下？年轻代和老年代有什么区别？
jvm内存模型分别为年轻代 和老年代  年轻代 一般存储的是 局部变量 这种使用过就删除的数据 。老年代就是 数据库连接池  bean对象 这些长时间存活的对象。

**知识点 33**：GC
**题**：Minor GC 和 Full GC 的区别？你项目怎么配 GC 参数的？
minor gc是年轻代的清扫器 清扫频率快 清扫速度也快。 full gc 是老年代清扫器，很久清扫一次，清扫的时候 系统会被暂停。我的项目应该是用默认的  没有具体配过参数
**知识点 34**：OOM
**题**：OOM 是什么？你项目中出现过吗？排查思路是什么？
oom是堆内存占用满了。我的项目中没有出现过，但是我知道如何进行排查。配置参数让 崩溃的时候 自动生成dump文件  再用mat工具 去查看哪个占用最多

**知识点 35**：栈和堆
**题**：堆和栈的区别？什么变量在栈上，什么在堆上？

栈存局部变量和方法参数，线程私有，方法结束之后就删除。堆是存new出来的对象，存活时间归gc管。一般局部变量在栈上，一般数据库连接池 或者bean对象在堆上

## 八、计算机网络

**知识点 36**：HTTP 和 HTTPS
**题**：HTTP 和 HTTPS 的区别？HTTPS 怎么保证安全的？

HTTP是明文传输，不安全。HTTPS是SSL+TLS加密层+HTTP,HTTPS用对称加密传数据，用数字证书校验身份

**知识点 37**：TCP 三次握手
**题**：TCP 为什么是三次握手，不是两次不是四次？

TCP是传输控制协议      http/https 是应用层   tcp是传输层    IP 是网络层

**知识点 38**：GET vs POST
**题**：GET 和 POST 的区别？POST 是安全的吗？

get和post的区别就是参数的位置不一样  get的参数拼接在url上，post的参数写在请求体上。post不保证安全，但是用https传输后数据也是安全的，但是不表示post本身是安全协议。

**知识点 39**：Cookie 和 Session
**题**：Cookie 和 Session 的区别？JWT 跟它们有什么关系？

cookie是存浏览器端，每次请求自动带上。session是存服务端，cookie里面只存session id 。jwt替代了seesion，由客户端去存放信息，服务端不存状态。

## 九、简历必问（从你项目里出）

**知识点 40**：项目难点
**题**：项目里遇到的最难的技术问题是什么？怎么解决的？
分布式锁和事务的顺序问题。我们一般是在操作数据库的业务代码外层添加@transactional注解，但是后面做高并发测试的时候发现 这样会有其他锁拿到旧数据导致停车场重复计费的情况。在代码中锁先被释放了，但是事务还没用提交，这就有一个空窗期 其他的抢到锁会读到旧数据。所以后面我用transactiontemplate 先拿到锁再去开启事务 事务提交了 再释放锁。

**知识点 41**：项目亮点
**题**：你觉得你项目里最能拿出手的一个亮点是什么？
最亮眼的可能是这个智能客服模块。做了混合检索（关键词+向量语义，加权合并），prompt拼装，deepseek调用，normalize输出校验，模型的回答可追溯，可验证。还做了无资料拒绝机制---检索不到资料就不调用大模型，再加上可观测日志，每次ai的调用和耗时都有记录。

**知识点 42**：技术选型
**题**：你的项目为什么用 RabbitMQ 而不是 RocketMQ？为什么用 Nacos 而不是 Eureka？
我的社区项目不需要那么大的消息量，万级的吞吐完全够了。rocketmq支持事务消息 但是部署起来 也需要更大的部署资源。nacos可以做服务的注册和配置中心，一个中间件可以做两件事，并且eureka现在已经不更新了

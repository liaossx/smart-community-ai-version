三、项目经验
--------------------------------------------------------------------------------

项目名称：智慧社区微服务管理平台
技术栈：Spring Boot + Spring Cloud Alibaba + MyBatis-Plus + Spring AI + 
        MySQL + Redis + RabbitMQ + DeepSeek API + Docker
项目介绍：面向社区物业管理场景的微服务项目，涵盖物业缴费、活动报名、停车管理、工单
处理等传统业务模块，并在此基础上独立设计实现了一个 AI 智能服务模块，将大模型能力
引入社区管理场景。

【业务模块】
1. 活动报名库存方案
   在活动报名场景中，原先直接依赖数据库扣减库存，高并发时容易出现锁竞争和超卖问题。
   改为 Redis 预扣库存 + RabbitMQ 异步落库的方式，把瞬时流量先扛在缓存和消息队列
   这一层，响应速度明显提升，超卖问题得到解决。

2. 停车计费幂等控制
   停车场道闸上报比较频繁，重复抓拍时存在重复计费风险。在订单处理接口中加入 Redis
   分布式锁做幂等控制，同时调整了加锁和事务开启的顺序，减少高并发时数据库连接被
   长时间占用的情况。

3. 缓存优化与系统稳定性
   对车位余量查询等高频接口使用 Spring Cache + Redis 做缓存处理，配合定时任务做
   缓存预热。压测时接口平均响应时间从 292ms 降到 3ms 以内。同时结合 Nacos 做配置
   动态调整，使用 Sentinel 做网关限流和服务降级，通过定时任务处理超时未支付订单，
   保证高并发下系统的稳定运行。

【AI 智能服务模块】（独立设计开发）
4. RAG 智能客服问答系统
   实现了关键字 + 向量语义的混合检索策略：先从 MySQL 知识库中检索与居民问题相关的
   社区资料（公告、维修电话、制度等），再将资料拼入 Prompt 交由 DeepSeek 生成回答。
   检索层设计了加权合并算法（关键字命中权重更高，向量辅助语义兜底），并加入"无资料
   拒绝"机制，避免模型在缺乏依据时编造回答。回答输出层通过 Normalizer 对 citation
   做白名单校验，确保每条引用来源真实可追溯。

5. 工单智能分析
   通过 System Prompt 约束模型输出结构化的工单分类结果（故障类别、优先级、风险等级、
   建议响应时间等），并在输出层设计 Normalizer 校验器：对模型返回的优先级做限幅校准，
   结合高危关键词（困人、起火、燃气泄漏等）和模糊词（不确定、好像等）做二次判断，
   降低误判和漏判风险。

6. 运营数据洞察与周报生成
   支持手动指标传入和自动数据库聚合两种模式。模型基于社区报修、投诉、欠费等运营
   指标，自动生成风险洞察卡片和可执行行动项。设计了 Fallback 降级策略：当大模型
   不可用时自动切换为基于业务规则的保底输出，确保运营页面不会出现空白。

7. 知识库管理与 AI 可观测性
   搭建了知识文档的 CRUD 管理体系，支持向量 Embedding 重建（可切换本地 Hash 演示
   模式与真实语义 Embedding），以及公告知识的定时同步。同时为每条 AI 调用链路设计了
   统一的日志记录模型（包含请求 ID、业务类型、调用耗时、置信度、成功/降级/失败
   状态），便于追踪每次大模型调用的成本和效果。
